Week Seven Technical Blog

SQL Injections

SQL Injection can destroy your database. It is a code injection technique where a user can insert, or inject, a SQL command into a SQL statement through a web page input. w3 schools gives an example of SQL injection where 1=1 is always true. A user can take a command and add or 1=1 to the end of it, so the statement will always return true, even if the other command is false.
A dangerous example..

SELECT UserId, Name, Password FROM Users WHERE UserId = 105 or 1=1

Now a hacker has the ability to have access to all users passwords. A hacker might get access to user names and passwords in a database by inserting " or ""=" into the user name or password text box.

Protection against SQL Injections

Some web developers use a "blacklist" of words or characters to search for in SQL input, to prevent SQL injection attacks but it isn't a very good idea since many of these words are used in common language, such as delete or drop. The only proven way to protect a website and prevent SQL injection is SQL parameters. SQL parameters are values that are added to an SQL query at execution time, in a controlled manner. Other than SQL parameters, the site should be coded in such a way to protect against data leaks.
