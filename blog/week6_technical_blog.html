Ruby Variables

local_variables

Are local in the code from which they are declared. If a variable is declared within a defined method, it cannot be accessed outside of that method.


$global_variables

Use a dollar sign before the constant. A global variable can be accessed anywhere within the ruby program. Their use is discouraged because they are visible anywhere within the program but even worse than can be changed from anywhere within the program. It makes debugging a program more difficult.


@@class_variables

Use two at (@) signs before the constant. Are shared with all instances of a class. Only one variable value exists for all objects for the class. If changing the value of the variable will change it throughout the class.


@instance_variables

Use one at(@) sign before the constant. Are similar to class variables, but their values are local to a specific instance of the object. For example, if the instance variable's value changes within a method, other methods will stay with their default value given.
