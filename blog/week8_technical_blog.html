Regular Expressions aka Regex or Regexp

A regular expression a special sequence of characters that helps you match or find strings.It uses a specialized syntax held in a pattern. Regular expressions are used for pattern matching and text processing.


Regexps are created using the...

/.../
%r{...}
Regexp::new

Some of the most common expressions are...


. Matches any single characters except a new line
* Matches 0 or more of the preceding character
+ Matches 1 or more of the preceding character
? Preceding character is optional. Matches 0 or 1 occurrence
\d Matches any single digit
\w Matches any word character (alphanumeric and underscore)
\s Matches any white space character
\S Matches any character that is not white space
\b Matches a word boundary position such as whitespace, punctuation, or the start/end of the string. This matches a position, not a character.
\B Matches any position that is not a word boundary. This matches a position, not a character.
{1,3} Matches the specified quantity of the previous token. {1,3} will match 1 to 3. {3} will match exactly 3. {3,} will match 3 or more.
[XYZ] Matches any single character from the character class
[XYZ]+ Matches one or more of any of the characters in the set
$ Matches the end of a string
^ Matches the beginning of a string
[^a-z] When ^ inside a character class, it means NOT
Example

Below, we are checking if a phone number has the correct amount of digits

/\(\d{3}\) \d{3}-\d{4}/

To check if it is a match we will use .match

"(123) 456-7890".match /\(\d{3}\) \d{3}-\d{4}/
/\(\d{3}\) \d{3}-\d{4}/.match "(123) 456-7890"

These will return true.
